# 경전회의 기사 대시보드 기획안

> 작성일: 2026-03-22
> 대상: 현대백화점그룹 홍보실 내부 운영용

---

## 1. 서비스 목적

매일 아침 경전회의 준비를 위해, 당일 수집된 유통업계 주요 기사를 섹션별로 자동 분류하여 제공하는 내부 모니터링 대시보드.

- **사용 패턴**: 담당자가 매일 오전 9시 이후 대시보드를 열면 당일 기사가 정리되어 있음
- **핵심 가치**: 수동 검색 없이, 분류된 기사를 바로 경전회의 자료로 활용 가능
- **보조 기능**: 중요 기사를 저장해 두고 언제든 다시 꺼내볼 수 있음

---

## 2. 데이터 흐름

```
[서버 시작 시 DB 감지]
  └─ DB 비어있음 → 최근 30일치 전체 수집 (초기 수집)
  └─ DB 있음 → 패스

[매일 09:00 자동 크롤링]
  └─ 전날 09:00 ~ 당일 09:00 발행 기사 수집
  └─ 해당 날짜(YYYY-MM-DD)로 DB 저장
  └─ 수동 진행 데이터는 건드리지 않음

[수동 수집 버튼 클릭]
  └─ 당일 09:00 ~ 클릭 시각 발행 기사 수집
  └─ "수동 진행" 레이블로 임시 저장
  └─ 화면은 "수동 진행" 뷰로 전환

[키워드 룰 기반 자동 분류]
  └─ 섹션1~4 배정 / 기타 제외

[DB 보관]
  └─ 수집일 기준 30일 보관 후 자동 삭제
  └─ "수동 진행" 데이터는 다음날 자동 크롤링 후 삭제
```

## 3. 수집 모드 3종

| 모드 | 트리거 | 수집 범위 | 드롭박스 표시 |
|------|--------|----------|--------------|
| **초기 수집** | 서버 시작 시 DB 비어있으면 자동 실행 | 최근 30일치 전체 | 30개 날짜 채워짐 |
| **자동 크롤링** | 매일 09:00 스케줄러 | 전날 09:00 ~ 당일 09:00 | `2026-03-22` 형식 날짜 |
| **수동 수집** | 기사 수집 버튼 클릭 | 당일 09:00 ~ 클릭 시각 | `수동 진행` |

## 4. 기사 분류 체계

분류는 키워드 룰 기반으로만 동작. 우선순위 순서대로 처음 매칭된 섹션에 배정.

| 우선순위 | 섹션 | 조건 |
|---------|------|------|
| 1 | 제외 (기타) | 제목에 무관 패턴 (증시, 날씨, 일정 등) |
| 2 | 섹션3 당사관련 | 제목에 당사 계열사명 포함 |
| 3 | 섹션2 반면교사 | 제목에 경쟁사명 + 본문에 위기키워드 |
| 4 | 섹션4 경쟁사관련 | 제목에 경쟁사명 (위기 없음) |
| 5 | 섹션1 유통트렌드 | 동일 업계 2개사 이상 등장, 또는 트렌드/정책 키워드 |
| 6 | 제외 (기타) | 위 어디에도 해당 없음 — DB 저장 안 함 |

> 분류 세부 기준은 `CLASSIFICATION_RULES.md` 참조

## 5. 화면 구성

### 5-1. 헤더 (상단 고정)

```
[경전회의 기사 대시보드]  현대백화점그룹 홍보실
[ 날짜 드롭박스 ▼ ]  [ 저장된 기사 보기 ]  [ 기사 수집 ]
```

- **날짜 드롭박스**: 최근 30일 날짜 + 최상단에 `수동 진행` (수동 수집 데이터가 있을 때만 표시)
  - 날짜 선택 시 해당 날짜 기사로 전환
  - 기사 수집 버튼 클릭 시 수동 수집 실행 후 드롭박스가 `수동 진행`으로 자동 전환
- **저장된 기사 보기**: 저장 기사 목록 페이지로 이동
- **기사 수집**: 수동 수집 실행

### 5-2. 통계 카드 (4개)

| 카드 | 내용 |
|------|------|
| 전체 기사 | 선택된 날짜(또는 수동 진행)의 총 기사 수 |
| 반면교사 | 섹션2 기사 수 |
| 당사 관련 | 섹션3 기사 수 |
| 경쟁사 동향 | 섹션4 기사 수 |

### 5-3. 섹션별 분포 바

섹션1~4 각각의 건수와 비율을 시각적으로 표시.

### 5-4. 섹션 필터 탭

전체 / ① 유통트렌드 / ② 반면교사 / ③ 당사관련 / ④ 경쟁사관련

### 5-5. 기사 카드

```
┌──────────────────────────────────────────────────┐
│ [기사 제목 링크]                                  │
│                                                  │
│ AI 메모 (분류 근거 한 줄)                         │
│                                                  │
│ [섹션 뱃지] [카테고리] [관련기업]   매체명         │
│                                                  │
│ 수집일자                          [ 기사 저장 ]  │
└──────────────────────────────────────────────────┘
```

- **매체명**: 기존 중요도(상/중/하) 위치에 표시
  - 네이버 뉴스 API에 기자명 필드 없음 → 매체명만 표시
- **기사 저장 버튼**: 저장 DB에 없으면 활성화 / 이미 저장된 기사는 `저장됨` 비활성화

## 6. 저장 기사 기능

### 목적
경전회의 자료로 쓸 기사를 별도 보관. 날짜 무관하게 누적.

### 저장 DB (saved_articles)
- articles DB와 독립된 테이블
- 날짜 제한 없이 영구 보관
- 사용자가 "비우기" 버튼을 누를 때만 전체 삭제

### 저장 기사 목록 페이지
헤더의 "저장된 기사 보기" 버튼으로 진입.

```
[ ← 돌아가기 ]                        [ 비우기 ]
────────────────────────────────────────────────
기사 제목 (링크)
기사 내용 요약
매체명 · 수집일자
[섹션 뱃지] [카테고리] [관련기업]
────────────────────────────────────────────────
(이하 반복, 저장 순서대로 누적)
```

## 8. 기사 그룹핑 기능 (유사 기사 묶기)

### 목적
동일 사건·보도자료에서 파생된 유사 기사를 묶어 한눈에 파악.

### 동작 방식

```
하루치 기사 수집 완료 (전날 09:00 ~ 당일 09:00)
        ↓
각 기사 제목에서 명사 추출 (KoNLPy + Okt)
  예) "신세계마켓 1주년 강남점 핵심 콘텐츠" → [신세계마켓, 1주년, 강남점, 콘텐츠]
        ↓
기사 쌍(pair)별 Jaccard 유사도 계산
  Jaccard = 교집합 명사 수 / 합집합 명사 수
        ↓
유사도 ≥ 0.35인 기사들을 같은 그룹으로 묶음
        ↓
group_id를 articles DB에 저장
```

### 날짜 독립성
- 그룹핑은 **해당 날짜 수집 범위 내에서만** 수행
- 2026-03-21 그룹과 2026-03-22 그룹은 완전히 독립
- 날짜가 달라지면 group_id 재부여

### 임계값 기준 (엄격)
| Jaccard 유사도 | 예시 |
|--------------|------|
| 0.35 이상 → 같은 그룹 | "신세계마켓 강남점 1주년" ↔ "신세계마켓 1주년 강남점 행사" |
| 0.35 미만 → 별개 기사 | "신세계마켓 1주년" ↔ "신세계 강남 매출 현황" |

### 프론트엔드 표시
```
┌─ [그룹] 신세계마켓 1주년 관련 기사 3건 ▶ 펼치기 ──────┐
│  (접힌 상태: 대표 기사 제목만 표시)                    │
└──────────────────────────────────────────────────────┘

펼쳤을 때:
┌─ [그룹] 신세계마켓 1주년 관련 기사 3건 ▼ 접기 ───────┐
│  ├ 기사① "장보기도 프리미엄"…신세계마켓 1주년, 강남점  │
│  ├ 기사② 신세계마켓 강남점 1돌…"그로서리 명소 자리매김"│
│  └ 기사③ 신세계마켓 1주년 맞아 할인 이벤트 진행        │
└──────────────────────────────────────────────────────┘
```

- 그룹 내 기사가 1건이면 일반 카드로 표시 (그룹 UI 미적용)
- 각 기사에 "기사 저장" 버튼 개별 적용

### 기술 스택 추가
- `KoNLPy` + `Okt` (Open Korean Text) — Java(JDK 1.7+) 설치 필요
- 그룹핑은 수집 완료 후 백엔드에서 일괄 처리
- articles 테이블에 `group_id` 컬럼 추가 (nullable, 그룹 없으면 null)
